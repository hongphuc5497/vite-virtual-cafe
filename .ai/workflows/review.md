# Review Workflow

## Code Review

Findings first:

```markdown
Severity:
File:
Problem:
Risk:
Fix:
```

## Product Review

OpenClaw should answer:

- Is this solving a painful workflow?
- Is scope too broad?
- What should be cut?
- What proof would make this worth continuing?

## Merge Gate

Do not call work complete until:

- Changed files are listed.
- Verification is listed.
- Known risks are listed.
- Follow-up is either created or intentionally ignored.
