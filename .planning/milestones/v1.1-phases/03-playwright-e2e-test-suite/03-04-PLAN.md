# Plan 03-04: Run full test suite, debug failures, achieve green suite

<objective>
Run the complete E2E test suite against the JS codebase, debug any failures, fix test issues or component bugs, and achieve a fully green test suite. All 4 user flows must pass.
</objective>

<must_haves>
- All 4 test files pass: `npx playwright test` exits 0
- No skipped tests, no `.skip()` calls
- Test suite runs in under 2 minutes
- Any component bugs discovered during testing are fixed
- Test reliability: running the suite 3 times in a row passes all 3 times
</must_haves>

<tasks>

<task number="1" goal="Run full test suite and collect failures">
<action>
Run the complete test suite:
```
npx playwright test
```

If all tests pass → skip to task 4.

If any tests fail, collect the failure output:
```
npx playwright test --reporter=line 2>&1 | tee test-output.txt
```

Categorize failures:
1. **Selector not found** → missing `data-testid` in component → fix component
2. **Timeout waiting for element** → timing issue → adjust timeout or add explicit wait
3. **Assertion failure** → test logic wrong or component behavior different than expected → investigate
4. **Audio/network error** → stubbing not working → fix route interception
5. **Build/dev server issue** → server not starting → fix config

For each failure, determine root cause and fix in subsequent tasks.
</action>
<read_first>
- All 4 test files (tests/timer.spec.js, tests/navigation.spec.js, tests/audio.spec.js, tests/relax.spec.js)
- All modified components for data-testid attributes
- playwright.config.js
- test-output.txt (after running)
</read_first>
<acceptance_criteria>
- Test suite runs (may have failures — this is the discovery task)
- Failure categories identified and logged
- `npx playwright test --reporter=list` produces output with test names visible
</acceptance_criteria>
</task>

<task number="2" goal="Fix test failures iteratively">
<action>
Work through failures one at a time, starting with the simplest:

**Selector not found:**
- Add missing `data-testid` attribute to the component
- Verify the attribute name matches what `tests/constants.js` expects
- Re-run the specific test: `npx playwright test tests/{file}.spec.js`

**Timeout waiting for element:**
- Check if element is conditionally rendered (loading state, suspended rendering)
- Add `waitFor({ state: 'attached' })` before interacting
- Increase timeout if animation-related (celebration fade-in)
- Verify the page loads correctly in non-test browser: `npm run dev` + manual check

**Assertion failure:**
- Read the component source to verify expected behavior
- Update test assertion to match actual behavior if test expectation was wrong
- Fix component if component behavior is wrong
- Check localStorage key names match between app and test constants

**Audio stubbing not working:**
- Verify route pattern matches actual MP3 URLs
- Check that route interception is set up BEFORE navigation
- Try broader pattern: `**/*.mp3*` or `**/*.mp3**`

**Build/server issues:**
- Verify `npm run dev` starts correctly on port 3000
- Check for port conflicts: `lsof -i :3000`
- Verify `webServer` config in playwright.config.js

After each fix, re-run the failing test to confirm it passes before moving to the next failure.
</action>
<read_first>
- The failing test file(s)
- The component(s) the failing test targets
- tests/constants.js (verify selector names)
</read_first>
<acceptance_criteria>
- Each fixed test passes individually: `npx playwright test tests/{fixed}.spec.js` exits 0
- No workarounds that skip assertions (no `.skip()`, no empty test bodies)
- Fixes are in components or test files (not both unless genuinely needed)
</acceptance_criteria>
</task>

<task number="3" goal="Ensure test reliability — 3 consecutive green runs">
<action>
Run the full suite 3 times consecutively:
```
npx playwright test && npx playwright test && npx playwright test
```

If any run fails, investigate the flaky test:
- Check for race conditions (did the test assume state from a previous test?)
- Add `test.beforeEach` to reset state between tests
- Ensure localStorage is cleared between tests
- Verify no shared state leaks between tests

Make tests independent:
- Each test should work in isolation
- Use `test.describe` serial mode only when truly needed
- Clear localStorage at the start of each test with `page.evaluate(() => localStorage.clear())`

All 3 runs must pass green. If a test flakes on run 2 or 3 but not run 1, it's not reliable.
</action>
<read_first>
- playwright.config.js (retries setting)
- All 4 test files
- Failing test output from flaky runs
</read_first>
<acceptance_criteria>
- 3 consecutive `npx playwright test` runs all exit 0
- No test uses `.skip()` or `.fixme()`
- `grep "test.describe.serial" tests/*.spec.js` returns empty (tests are independent)
- No test output shows "flaky" or "unreliable" indicators
</acceptance_criteria>
</task>
