## Animation Contract

### Panel Exit (session start)

| Element | Property Change | Duration | Easing |
|---------|----------------|----------|--------|
| Left column (SessionTimer card + VibeSelector card) | opacity: 1 -> 0, translateX: 0 -> -40px | 400ms | ease-out |
| Right column (RoomMixControls card) | opacity: 1 -> 0, translateX: 0 -> 40px | 400ms | ease-out |
| Hero text (h1 + subtitle) | opacity: 1 -> 0, translateY: 0 -> -20px | 400ms | ease-out |

All three animate simultaneously. After animation completes, set `pointer-events-none` to prevent interaction with invisible elements. Elements remain in the DOM (never conditionally removed) to prevent React instant-disappearance.

### Timer Crossfade

| Element | Start | Duration | End |
|---------|-------|----------|-----|
| Card-based SessionTimer | opacity: 1 | 300ms | opacity: 0 |
| Floating ImmersiveTimerPill | opacity: 0 (start at ~100ms delay) | 300ms | opacity: 1 |

Two separate elements, not a single animated element. The pill fades in with ~100ms delay after panels begin their exit animation, creating a staggered crossfade. Pill uses `fixed` positioning (centered viewport) so it escapes any parent `overflow-hidden` container.

### Panel Reveal (Escape / edge-hover)

Exact reverse of the exit animation -- same 400ms ease-out. Opacity returns to 1, translate returns to 0. Pill remains visible during reveal -- it does not crossfade back to the card timer.

### Panel Auto-Hide (3s inactivity)

Same animation as panel exit. Pill remains visible.

### Timer End (session complete)

When `timer.timeLeft === 0` and `timer.isRunning === false`:
1. `showCelebration = true` fires (existing effect)
2. Panels restore to default visible state (reverse animation, no auto-hide)
3. ImmersiveTimerPill fades out (300ms ease-out)
4. CelebrationOverlay appears (z-50, above panels)

**Source:** CONTEXT.md D-01, D-02, D-03, D-06, D-07 (all locked decisions)

## Interaction Contract

### Enter Condition
- `timer.isRunning` transitions from `false` to `true`
- Two-phase approach: first set `isRunning` (timer starts), then in a subsequent `useEffect` triggered by `isRunning`, set `isActive=true` to trigger the CSS transition. This prevents animation stutter by ensuring the browser paints the "running" state before applying transition classes.

### Exit Condition
- `timer.isRunning` transitions to `false` AND `timeLeft === 0` (session complete)
- `exitImmersiveMode()` called, `isActive=false`, `showPanels=true`
- New session start re-enters immersive mode

### Escape Key
- Toggles between panels-visible and panels-hidden during immersive mode
- Escape handler lives in `useImmersiveMode` (separate from `useKeyboardShortcuts`)
- **Guard:** Check `showCelebration` before processing Escape to avoid conflict with celebration overlay dismiss
- Does NOT pause/stop the session

### Edge-Hover Detection
- Transparent `fixed` div at bottom 10% of viewport (`style={{ height: "10vh" }}`)
- `onMouseEnter` calls `revealPanels()`, starting auto-hide inactivity timer
- `aria-hidden="true"` for screen reader accessibility
- Research recommends CSS `onMouseEnter` on React element over JS `mousemove` listener (fewer events, zero overhead when cursor is not near edge)

### Auto-Hide Inactivity Timer
- When panels are revealed and cursor is not in bottom 10% zone:
  - 3-second debounced timeout starts
  - Mousemove listener resets the timeout
  - If 3 seconds pass without mousemove outside edge zone, `showPanels=false`
- Mousemove listener only active when `showPanels=true && isActive=true`
- Cleaned up on unmount via `useEffect` return

### Keyboard Shortcuts During Immersive Mode
- Space bar (toggle play/pause) stays active -- users can pause without revealing panels
- Arrow keys (volume control) stay active
- Escape is exclusively for toggling panel visibility (handled by `useImmersiveMode`, not `useKeyboardShortcuts`)
- Rationale: consistent with audio player behavior (Space works in fullscreen/cinema mode)

### Reduced Motion
- Check `window.matchMedia("(prefers-reduced-motion: reduce)").matches`
- When true: skip transition classes, panels appear/disappear instantly
- gated in `useImmersiveMode` via `useEffect`

### Accessibility
- `aria-hidden="true"` on hover strip
- `role="timer"` on the pill `aria-label="Remaining time: [formatted time]"`
- Panels remain in DOM when hidden (opacity 0 + pointer-events-none) -- screen readers do not see them
- Focus management: TAB focus should be trapped during immersive mode or users can TAB to invisible panels. Consider `inert` attribute on hidden panels.

**Source:** CONTEXT.md D-06 (reveal), RESEARCH.md (edge detection, auto-hide, keyboard patterns, anti-patterns), `useKeyboardShortcuts.js` (existing pattern)
