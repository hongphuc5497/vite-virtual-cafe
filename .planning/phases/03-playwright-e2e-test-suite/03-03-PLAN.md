<task number="2" goal="Write tests/audio.spec.js">
<action>
Create `tests/audio.spec.js` with audio mixer E2E tests.

Use audio stubbing (same silent MP3 route interception as 03-02).

Test cases:

**Test 1: Tracks can be played and paused individually**
1. Navigate to `/`
2. Find a track's play button, e.g., `[data-testid=track-play-rain]`
3. Click play — verify the button state changes (pause button becomes visible)
4. Click pause — verify it returns to play state
5. Repeat for a second track to verify independent control

**Test 2: Volume sliders adjust output**
1. Navigate to `/`
2. Find a track's volume slider: `[data-testid=track-volume-rain]`
3. Get initial volume value from the slider
4. Change the slider value (drag or set via JavaScript)
5. Verify the new value differs from initial
6. Verify the value persists after page reload via localStorage check

**Test 3: Playback failure shows error badge with retry**
1. Navigate to `/`
2. Intercept a specific MP3 route to return 500 error: `page.route('**/rain.mp3', route => route.abort())`
3. Trigger play on that track
4. Verify error badge appears: `[data-testid=track-error-rain]` becomes visible
5. Click retry button: `[data-testid=track-retry-rain]`
6. Verify retry attempts playback

**Test 4: Selecting a vibe preset changes active track set**
1. Navigate to `/`
2. Note which tracks are visible/active
3. Click a vibe button: `[data-testid=vibe-Lo-fi]`
4. Verify the UI reflects the vibe selection (button highlighted, tracks updated)

Use `test.describe('Audio Mixer', () => { ... })`.
</action>

<task number="3" goal="Write tests/relax.spec.js">

1. Navigate to `/relax`
2. Find play controls within the ambient player
3. Verify tracks can start playing (button state changes)
4. Verify no timer-related UI is present

**Test 3: Paused tracks state restored from localStorage**
1. Set up localStorage with a known pausedTracks state BEFORE navigating:
   ```js
   await page.evaluate(() => {

   });
   ```
2. Navigate to `/relax`
3. Verify the tracks 'rain' and 'fireplace' appear in paused state (pause buttons visible instead of play)
4. Clear localStorage and reload — verify default state

Use `test.describe('Relax Page', () => { ... })`.
