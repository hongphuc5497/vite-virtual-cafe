<task number="2" goal="Write tests/timer.spec.js">
<action>
Create `tests/timer.spec.js` with the focus timer E2E tests.

Use audio stubbing via `page.route()` to intercept all MP3 requests and return a short silent audio blob:
```js
const silentAudio = Buffer.from(/* minimal WAV header + silence */);
await page.route('**/*.mp3', (route) => route.fulfill({
  body: silentAudio, contentType: 'audio/mpeg'
}));
await page.route('**/*.wav', (route) => route.fulfill({
  body: silentAudio, contentType: 'audio/wav'
}));
```

Test cases:

**Test 1: Timer countdown updates in real time**
1. Navigate to `/`
2. Click the 25-minute preset button (`[data-testid=preset-25]`)
3. Click start (`[data-testid=timer-start]`)
4. Wait 1-2 seconds, then check that timer display text changes (e.g., starts with "24:5")
5. Use `expect(timerDisplay).toContainText(/24:5/)` with timeout matching TIMER_TICK

**Test 2: Timer reaches zero → celebration appears**
1. Set a very short duration if possible, or use a custom 1-minute timer
2. Start the timer
3. Wait for celebration overlay to appear: `page.locator('[data-testid=celebration-overlay]').waitFor({ state: 'visible', timeout: 120000 })`
4. Verify heading is visible with congratulatory text
5. Verify celebration has fade-in animation (check opacity/visibility class)

**Test 3: Session entry written to localStorage**
1. After celebration appears, check localStorage
2. `const sessions = JSON.parse(await page.evaluate(() => localStorage.getItem('focusSessions')))`
3. Assert sessions is an array with at least one entry
4. Assert the last entry has keys: `date`, `duration`, `vibe` (or equivalent fields)

Use `test.describe('Focus Timer', () => { ... })` to group tests.
</action>

<task number="3" goal="Write tests/navigation.spec.js">
<action>
Create `tests/navigation.spec.js` with navigation and scene E2E tests.

Use the same audio stubbing setup as the timer test.

Test cases:

**Test 1: Route switching between / and /relax**
1. Navigate to `/`
2. Verify main page elements are visible (timer display, mixer panel)
3. Click `[data-testid=nav-relax]` to go to /relax
4. Verify URL is `/relax` and ambient player is visible
5. Verify timer display is NOT visible on relax page
6. Click `[data-testid=nav-home]` to return to `/`
7. Verify timer display is visible again

**Test 2: Scene selector changes background**
1. Navigate to `/`
2. Click a scene button, e.g., `[data-testid=scene-library]`
3. Verify the page background or overlay class changes (check for CSS class or style change)
4. Click another scene, verify it changes again

**Test 3: Keyboard shortcuts**
1. Navigate to `/`
2. Press Space key: `await page.keyboard.press('Space')`
3. Verify audio play/pause state changes (check toggle state or button state)
4. Press ArrowUp key: `await page.keyboard.press('ArrowUp')`
5. Verify volume indicator or state reflects change

Use `test.describe` blocks: 'Navigation' and 'Keyboard Shortcuts'.
</action>
